Kapitel 1 — Einstieg

7-Level Skill-Ladder

LEVEL 1
Prompter
Chat als Suchmaschine. Erste Antwort wird genommen, wie sie kommt. Kein Iterieren.
LEVEL 2
Conversationalist
Echte Dialoge: iteriert über Ergebnisse, gibt Feedback, liefert bewusst Kontext mit.
▲ Grösster Sprung: Features entdecken
LEVEL 3
Power User
System Prompts, Projects, Datei-Uploads, Code Execution gezielt eingesetzt.
LEVEL 4
Workflow Designer
Workflows statt Einzelprompts. Skills und Custom Instructions als wiederverwendbare Bausteine.
▲ Paradigmenwechsel: Nutzer wird Architekt
LEVEL 5
System Builder
Projects mit Skill-Architektur, Custom GPTs mit strukturiertem Wissen, automatisierte Pipelines.
LEVEL 6
Agentic Operator
Claude Code, MCP-Server, Subagents für mehrstufige autonome Aufgaben.
LEVEL 7
Platform Architect
Team-Templates, Governance-Frameworks, Schulungskonzepte. Baut KI-Systeme für andere.

Ziel der meisten Kurse: Von Level 1 auf Level 3 — mit klarem Ausblick auf Level 4.

Die Modellfamilie

Höchste Intelligenz und Tiefe. Ideal für komplexe Analysen, Forschung, mehrstufige Aufgaben und anspruchsvolles Coding.

Sonnet 4.6 — Ausgewogen

Bester Kompromiss aus Geschwindigkeit und Qualität. Standard für die meisten Alltagsaufgaben und Programmierung.

Haiku 4.5 — Schnell & effizient

Schnellste Antworten, günstigster Preis (API). Für einfache Aufgaben, Zusammenfassungen und schnelle Abfragen.

Modellwahl im Chat — Im Chatfenster kann das Modell oben links per Dropdown gewechselt werden. Pro-User haben Zugang zu allen Modellen inkl. Opus. Free-User nutzen Sonnet (mit Limit) und Haiku als Fallback.

Kapitel 4 — Kontext & Pflege

Wie pflegt man das Context Window?

Drei Strategien — sequenziell oder kombiniert.

↺

Reset

Neuer Chat. Lädt nur das nötigste neu — Anweisung, vielleicht ein Anhang. Schneller Schnitt, wenn die Konversation zu sehr abgeschweift ist.

Wann: wenn die letzten Antworten merklich schwächer werden.

∑

Summarize

Mittendrin um eine Zusammenfassung bitten („Fasse den bisherigen Stand zusammen"). Diese in einen neuen Chat kopieren — komprimiert viel Kontext in wenig Tokens.

Wann: bei langen Recherche- oder Brainstorming-Strängen.

⤴

Progressive

Nur das einbringen, was für die aktuelle Frage wirklich nötig ist. Kernkontext kurz, Details erst auf Nachfrage. Skills nutzen genau dieses Prinzip auf System-Ebene.

Wann: als Default-Haltung bei jeder neuen Aufgabe.

Kapitel 6 — Skills

Was ist ein Skill?

Was ist ein Skill? Ein Skill ist ein Ordner mit Instruktionen (primär eine SKILL.md-Datei), der Claude beibringt, wie bestimmte Aufgaben oder Workflows zu erledigen sind.

Testing. Drei Ebenen — Trigger-Tests (lädt der Skill zum richtigen Zeitpunkt?), Funktionale Tests (stimmt der Output?) und Performance-Vergleiche (besser als ohne Skill?).

Distribution. Skills werden als ZIP über Claude.ai hochgeladen oder im Claude Code Skills-Verzeichnis abgelegt.

Skill-Architektur: progressive disclosure — Drei Ebenen minimieren Token-Verbrauch bei maximaler Expertise
Ebene 1 — YAML frontmatter
Immer im System-Prompt geladen
name (kebab-case) · description (Was + Wann) · metadata (author, version…)
↓ Skill relevant? → laden
Ebene 2 — SKILL.md body
Vollständige Instruktionen, Steps, Beispiele, Troubleshooting
↓ Bei Bedarf navigieren
Ebene 3 — Verlinkte Dateien
scripts/ · references/ · assets/
