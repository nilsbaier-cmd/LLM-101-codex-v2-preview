Skills · Progressive Disclosure

Drei Stufen — nur das Nötigste auf einmal

Skills sind nicht alle gleich teuer. Claude lädt sie in drei Stufen — nur das Nötigste auf einmal.

Token-Verbrauch aktuell: ~50 nur die YAML-Frontmatter ist geladen
1
YAML-Frontmatter
~50 Tokens
Was Claude immer sieht:
name: aktennotiz-helper
description: Erstellt Aktennotizen
  im Hausstil. Aktiv wenn der User
  eine Aktennotiz, Memo oder Notiz
  schreiben möchte.
version: 1.0

Diese Metadaten sind permanent im System-Prompt — bei jeder einzelnen Claude-Anfrage. Daher müssen sie sehr kompakt sein. Aus der description entscheidet Claude, ob der Skill für die aktuelle Aufgabe relevant ist.

↓ Bei Relevanz
2
SKILL.md Body
~2000 Tokens
Wird geladen, wenn description matched:
# Aktennotiz-Helper

## Wann verwenden
- Wenn Nutzer „Aktennotiz", „Memo" oder „Notiz" erwähnt
- Wenn ein Sachverhalt strukturiert dokumentiert werden soll

## Output-Format
1. Datum (TT. Monat JJJJ)
2. Sachverhalt (1–2 Sätze, sachlich)
3. Beurteilung (2–3 Sätze)
4. Massnahmen (nummerierte Liste mit Fristen)

## Tonfall-Regeln
- Aktiv statt passiv
- Keine Floskeln
- ...

Der Body enthält konkrete Anweisungen, Beispiele, Format-Regeln. Wird nur geladen, wenn Claude den Skill aktiviert.

↓ Bei Bedarf
3
Verlinkte Dateien
variabel
Werden nachgeladen wenn die SKILL.md sie referenziert:
refs/hausstil-glossar.md (~800 Tokens)
refs/beispiele.md (~1200 Tokens)
scripts/format-validator.py (~500 Tokens)

Diese Dateien werden erst gelesen, wenn die SKILL.md sie aktiv referenziert (z.B. „siehe refs/hausstil-glossar.md"). Sonst bleiben sie verfügbar, aber kosten keine Tokens.

Effizienz: Skill vs. „Alles im System-Prompt"

Ohne Skills
~4000 Tokens
jede Anfrage trägt den vollen Hausstil + Beispiele + Validierer mit, auch wenn der Nutzer was ganz anderes will
Mit Skills
~50 Tokens
nur die YAML-Frontmatter — der Rest wird bei Bedarf nachgeladen, je nach Aufgabe

Faustregel — Skill-Design = Disziplin in der description. Klingt sie wie ein Trigger, wird sie zum Trigger. Kürzer ist besser.
